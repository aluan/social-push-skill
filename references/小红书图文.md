## 上传图文 workflow

1. 打开发布图文的网站：`agent-browser --headed --profile /tmp/agent-profile open "https://creator.xiaohongshu.com/publish/publish?source=official&from=tab_switch&target=image"`
2. 查看交互：`agent-browser snapshot -i`
3. 提取上传按钮 ref：从 snapshot 中找到文件上传元素（`input[type=file]` 或 `button` 包含"上传"）
4. 上传图片：`agent-browser upload @{上传按钮ref} "{图片路径}"`
5. 等待上传完成（约 2 秒）
6. 查看交互：`agent-browser snapshot -i`
7. 提取标题输入框 ref：从 snapshot 中找到 `textbox` 含"标题"关键字的元素
8. 插入标题（不超过 20 字）：`agent-browser fill @{标题ref} "{标题}"`
9. 插入正文：`agent-browser fill ".ProseMirror" "{正文内容}"`
10. 添加话题：`agent-browser type ".ProseMirror" " #{话题}"`，然后 `agent-browser press "Enter"` 确认
11. 查看交互：`agent-browser snapshot -i`
12. 提取发布相关按钮 ref：从 snapshot 中找到 `button "暂存离开"` 和 `button "发布"`
13. 最后一步根据用户要求选择：
    - 暂存草稿离开：`agent-browser click @{暂存离开ref}`
    - 发布（需用户明确确认）：`agent-browser click @{发布ref}`

# 注意
标题不要超过 20 字符

## 元素参考

### 发布页面（上传图片后）

从 snapshot 动态提取 ref，勿使用硬编码。典型元素如下：

| 功能 | grep 匹配模式 | 备注 |
|------|--------------|------|
| 上传图片 | `input` 或含"上传"的按钮 | 页面初始状态 |
| 标题框 | `textbox ".*标题.*"` | 找第一个含"标题"的 textbox |
| 正文框 | `.ProseMirror` | CSS 选择器，直接使用 |
| 暂存离开 | `button "暂存离开"` | 发布区域 |
| 发布按钮 | `button "发布"` | 发布区域 |

### ref 提取示例

```bash
# 提取标题输入框
TITLE_REF=$(grep 'textbox ".*标题.*"' /tmp/xhs_snapshot.txt | head -1 | sed -n 's/.*\[ref=\(e[0-9]*\)\].*/\1/p')

# 提取暂存离开按钮
DRAFT_REF=$(grep 'button "暂存离开"' /tmp/xhs_snapshot.txt | head -1 | sed -n 's/.*\[ref=\(e[0-9]*\)\].*/\1/p')

# 提取发布按钮
PUBLISH_REF=$(grep 'button "发布"' /tmp/xhs_snapshot.txt | head -1 | sed -n 's/.*\[ref=\(e[0-9]*\)\].*/\1/p')
```
